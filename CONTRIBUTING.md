# Contributing

Thanks for your interest in improving Awesome Login.

## Development Setup

```bash
bun install
bun dev
```

## Before You Open a Pull Request

Run the required checks:

```bash
bun run build
bun run lint
```

## Contribution Guidelines

- Keep the project focused on reusable login page UI patterns.
- Prefer small, reviewable pull requests.
- Preserve the lightweight stack unless a dependency brings a clear benefit.
- If you change the visual design, include a short note explaining the intent.
- If you change copy, keep the project public-facing and template-friendly.

## Commit Style

Use short, descriptive commit messages that explain the user-facing change.

Examples:

- `Refine login card spacing`
- `Improve animated character timing`
- `Document Vercel deployment`

## Pull Request Notes

Please include:

- what changed
- why it changed
- screenshots or recordings for visible UI updates
- any follow-up ideas that should stay out of the current PR
